# 快速开始

这份文档面向第一次安装和日常使用 Signal 的用户。

## 1. 安装前准备

| 项目 | 建议 |
|---|---|
| 系统 | macOS 10.15 及以上 |
| 安装包 | 不确定芯片架构时使用 `packages/macos/Signal_0.1.2_universal.dmg` |
| 分析引擎 | 推荐本机 Claude CLI，也支持本机 Codex CLI / TRAE CLI / Anthropic API |
| 数据目录 | 建议每个工作区使用一个独立 `brain` 目录 |
| 可选工具 | Git、Obsidian、飞书机器人 Webhook |

应用默认优先调用本机已登录的 CLI，不要求在应用内保存 API key。远程 API 模式适合无头机器或需要统一额度的环境。

## 2. 安装 macOS 包

1. 下载 `.dmg`。
2. 双击打开安装镜像。
3. 把 `Signal.app` 或 `Signal Brain.app` 拖到 `Applications`。
4. 从“应用程序”打开。
5. 首次打开如果被 Gatekeeper 拦截，在 Finder 中右键应用，选择“打开”。
6. 如果提示应用“已损坏”，在终端执行：

```shell
xattr -cr "/Applications/Signal.app"
# 或
xattr -cr "/Applications/Signal Brain.app"
```

## 3. 首次启动配置

首次打开会进入设置向导：

1. 选择或创建认知库目录。
2. 选择分析引擎：`Claude`、`Codex`、`TRAE` 或远程 API。
3. 填写关注方向，例如 `AI算力, HBM, 半导体设备, 电子特气`。
4. 保存后应用会初始化认知库结构和本地索引。

认知库目录是核心资产，里面保存 source、认知卡片、因子、图谱、报告运行记录等数据。SQLite 索引和报告可以重建，原始 source 与认知卡片才是事实源。

## 4. 推荐日常流程

1. 打开“今日”，先看待复核、今日热点、认知健康和需要回访的弱认知。
2. 到“投喂”粘贴电报、研报摘录、文件或 URL，再运行每日消化。
3. 到“热点”“大事”“商品”“DRAM”“财报库”等模块刷新外部线索。
4. 对高价值主题进入“分析报告”或“产业链拓扑”做深度研究。
5. 在“认知库”回看证据链，在“复核队列”处理低可信或高风险动作。
6. 把可交易假设登记到“事件因子”，等观察窗口满后验证。
7. 最后回到“决策就绪”，只把高质量、低矛盾、证据足够的认知作为辅助输入。

## 5. 工作区和数据隔离

应用支持多个工作区。每个工作区有自己的配置和认知库路径，切换工作区会重新挂载所有视图，避免不同工作区数据混用。

建议：

- 不同团队、策略或研究方向使用独立工作区。
- 不要让两个工作区指向同一个认知库目录。
- 需要备份时，对认知库目录单独配置 Git remote。

## 6. 可选 CLI 用法

桌面应用覆盖主要流程。熟悉命令行时，也可以使用源码仓库构建出来的 `signal` CLI：

```shell
signal init --vault ~/brain
signal feed 今日电报.md --vault ~/brain
signal digest --vault ~/brain
signal ask "六氟化钨供需怎么看" --vault ~/brain
signal auto "AI算力" --vault ~/brain
signal factor pending --vault ~/brain
```

CLI 支持投喂、消化、挖掘、热点、大事、基本面、商品、DRAM、持仓、个股技术分析、涨停、自动闭环、维护、同步、统计、问答、拓扑和财报库等能力。

## 7. 常见问题

### 提示没有配置认知库路径

进入“设置”或重新打开设置向导，填写一个独立目录并保存。

### 找不到 Claude / Codex / TRAE 命令

先在终端确认对应 CLI 已安装并登录。图形应用的 PATH 可能与终端不同，安装后重启应用。

### 每日消化提示没有新投喂源

当天还没有新 source。先到“投喂”添加文本、文件或 URL，再运行每日消化。

### 自动挖掘没有按时执行

检查应用是否运行、设置里的任务是否启用、当前工作区是否正确、并发上限是否被占满。

### 飞书日报没有推送

检查“设置”里的飞书机器人 Webhook 是否正确，确认机器人仍在目标群内，网络可以访问飞书接口。
