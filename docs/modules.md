# 模块使用说明

以下说明按应用左侧导航和源码中实际页面组织。

## 全局能力

### 工作区

工作区用于隔离配置和认知库数据。切换工作区后，今日、投喂、认知库、持仓、因子、拓扑等视图都会重新加载当前工作区的数据。

### 全局搜索

顶部搜索支持 `Cmd/Ctrl + K` 打开，可搜索认知卡片、热点、财报、分析任务、拓扑、会话和因子。点中结果后会跳到对应模块或打开抽屉。

## 今日

“今日”是每日回访入口，汇总认知库规模、认知健康分、待复核、待回访、到期因子、今日热点和今日新增认知。

常用操作：

- 点“维护”让衰老/停滞认知进入待处理。
- 点“同步”把认知库 Git 数据推送到已配置的 remote。
- 从热点、复核、因子、认知列表进入对应模块。

## 探索

“探索”提供主题式外部检索和候选知识沉淀。输入问题或方向后，系统会启动探索会话，返回候选发现；用户选择可信条目后再沉淀入认知库。

适合：

- 对一个新主题做初筛。
- 在已有认知不足时补齐外部候选。
- 把探索结果转成可追溯 source 和认知卡片。

## 热点

“热点”每日挖掘新闻并做来源真实性、影响甄别和个股关联。

常用操作：

- “挖掘热点”：按配置方向抓取当天热点。
- “导入消息”：导入群聊、热榜或消息文本，做热点解析。
- 查看可信度、来源层级、印证数量、涉及标的和受益链。
- 对单条热点发起探索、深度分析或财报库关联。

过闸热点会沉淀为认知卡片，低可信或仅模型推断的条目应进入人工复核。

## 大事

“大事”前瞻挖掘未来约两个月可能影响市场的重要事件，并生成影响分析与情景预案。

视图模式：

- 日历：按日期看事件密度。
- 时间轴：按时间顺序浏览。

常用操作：

- “刷新大事”重新挖掘前瞻事件。
- 点击事件查看影响方向、重要度、涉及资产和情景预案。
- 对重要事件发起深度报告。

## 财报

“财报”围绕关注列表挖掘历史定期报告，抽取财务序列、关键确认点和可沉淀认知。

常用操作：

- 按监听组跑财报挖掘。
- 查看财报条目、财务序列和已沉淀卡片。
- 打开原文链接或进入详情分析。

## 财报库

“财报库”构建全市场公司基本面和概念索引，用于按热点/概念召回个股，并在抽屉中查看公司财报分析。

常用操作：

- 构建或增量刷新财报库。
- 选择市场：A 股、美股、日股或全部。
- 用“概念召回”“全文检索”“代码/名称查找”定位公司。
- 打开“财报分析”查看 KPI、营收/净利/毛利率趋势、财务表和本地解读。
- 从热点反查相关公司，或对候选公司发起深度研报。

## 研报

“研报”汇总财报确认、价格变化和行业研报，把已挖掘素材交给分析引擎生成深度投研报告。

常用操作：

- 跑全部监听或指定监听。
- 查看财报确认、涨价线索、行业研报和已沉淀认知。
- 一键生成深度研报，进入“分析报告”查看进度和成稿。

## 商品

“商品”追踪大宗商品现货/期货价格，按 1 个月、3 个月、6 个月、1 年、YTD 等跨度排序，并建立“上游现货涨价到个股”的传导链。

常用操作：

- “刷新榜单”抓取真实价格并生成涨跌榜。
- 切换排序跨度和品类筛选。
- 查看领涨品种、来源、传导链和相关个股。
- 对某个品种发起商品传导链深度分析。

## 存储 / DRAM

“DRAM”抓取公开 DRAM 价格快照，合成存储股价格、宏观/财报事件和相关性分析。

常用操作：

- 刷新最新 DRAM 快照。
- 切换历史快照。
- 查看 DRAM 现货走势、相关股票 K 线、事件窗口、股价与预测。
- 点“深度分析”生成存储周期报告。

## 涨停

“涨停”抓取 A 股涨停/炸板池，匹配财报库并计算持续性。

常用操作：

- 设置日期区间并刷新。
- 按连板数、题材性/未证、财报匹配关系筛选。
- 查看共性因子、次日封板率、3 日内再封板率。
- 打开公司财报分析，判断题材是否有主营或财报佐证。

## 拓扑

“产业链拓扑”把主题拆成上游、中游、下游，并对环节和标的做证伪裁决。

常用操作：

- 输入产业链或主题，例如 `AI算力`、`固态电池`、`CPO 光互联`。
- 点“测绘拓扑”生成结构化链路。
- 在抽屉中查看每个节点的供需紧张度、目标公司、证伪结论和构建过程。
- 对已有拓扑重新证伪或删除。

## 分析报告

“分析报告”基于共享认知库和 `signal-analyst` 框架创建深度报告。任务后台运行，可点开查看进度、成稿和引用。

常用操作：

- 输入方向/主题创建任务，留空表示全库扫描。
- 查看运行中、已完成、失败任务。
- 对报告做校准、追问、重试、导出 HTML。
- 当认知覆盖不足时，系统会补充外部发现并标记为待核验。

## 认知库

“认知库”是核心资产浏览器，用于查看已沉淀的认知卡片。

常用操作：

- 按标题、论点、证据检索。
- 按状态筛选：全部、活跃、衰老、已证伪、已合并。
- 打开卡片查看 thesis、evidence、hypotheses、changelog。
- 查看来源可信度、相关标的和趋势。
- 对低质量、衰老或待补证据条目做回访处理。

## 自动挖掘

“自动挖掘”执行完整闭环：挖掘素材、每日消化、评测、写入认知库、生成日报。

常用操作：

- 点“立即自动挖掘”按默认方向运行。
- 输入临时方向，例如 `电子特气`，手动跑一次。
- 在“设置”配置多个定时任务、周期、触发时间和最大并发数。
- 查看每个任务的最近状态、均分、创建/更新/丢弃数量。

## 投喂

“投喂”是原料入口，支持文本、文件、URL。

常用操作：

- 粘贴电报、研报摘录、公众号文章或群聊记录。
- 拖拽文件到投喂区，或点击选择文件。
- 输入 URL 抓取网页内容。
- 点“运行每日消化”，把当天 source 转成认知卡片。

系统按内容 SHA-256 去重，重复资料不会重复写入。

## 持仓

“持仓”管理组合并生成盘前预测、盘中策略、盘后复盘、热点推送和深度研报。

常用操作：

- 新建/切换/重命名/删除持仓组合。
- 导入持仓文本或截图识别持仓。
- 刷新行情，查看组合盈亏。
- 生成盘前、盘中、盘后、热点或深度报告。
- 对单只持仓打开财报、产业链拓扑、技术面和深度研报抽屉。

## 资金

“资金流向”按市场追踪板块和个股资金流，区分主力、机构、大户和散户净流入。

常用操作：

- 切换市场。
- 搜索个股或板块。
- 查看资金净流入、排名、来源和明细。
- 用资金流方向辅助识别行情驱动，但不要单独作为交易依据。

## 个股

“个股”提供 A 股、美股、港股的 K 线和技术面分析。

常用操作：

- 输入 `AAPL`、`600519.SH`、`00700.HK` 等代码查询。
- 切换分时、5 日、日 K、周 K、月 K。
- 切换区间和副图：VOL、MACD、KDJ、RSI。
- 查看确定性指标核计算的趋势、动能、波动、量价、关键价位和综合评分。
- 点 AI 研判时，AI 只解读已计算好的指标，不改动数值。

## 期权

“期权”面向美股期权研究，包含 Black-Scholes 计算器、策略优化、最优买入和组合回测。

子模式：

- 计算器：输入标的、行权价、期限、波动率等参数计算 Greeks 和理论价格。
- 最优策略：输入方向、波动率、期限、资金和风险偏好，生成候选策略并画到期损益图。
- 最优买入：扫描真实期权链，找收益比更优的买入合约。
- 组合回测：组合正股、Call、Put 做历史情景验证。

## 图谱

“图谱”展示认知、来源、产业链、标的之间的关系，可重建 Obsidian 兼容图谱。

适合：

- 查看某个主题的证据网络。
- 发现同源放大、孤立认知或过度集中的引用。
- 从节点跳回认知卡片。

## 因子

“事件因子”把认知里的可验证假设登记为事件因子，用事件研究法验证相对基准的累计超额收益，并反哺源认知。

常用操作：

- 选择认知卡片、假设、标的、基准和观察窗口登记因子。
- 查看 pending、validated、falsified 等状态。
- 观察窗口走满后点验证。
- 删除无效因子或回看验证结论。

## 决策

“决策就绪”聚合高质量认知，按信念、证据可信度、来源多样性和内部矛盾打分。

常用操作：

- 查看认知健康分和决策候选。
- 识别方向矛盾、同源放大、低可信和停滞认知。
- 把矛盾对登记进复核队列。
- 打开候选认知查看证据链。

## 复核

“复核队列”处理高风险或低可信的认知动作。

常用操作：

- 切换待复核和全部记录。
- 批准可信动作。
- 否决仅模型推断或证据不足的动作；否决会保留历史，并可把相关认知置为已证伪。

## 问答

“问答”在启用时会用本机 Claude/Codex/TRAE 会话实时渲染思考、工具调用和最终回答。

常用操作：

- 输入问题并提交。
- 查看消息流、工具事件、引用认知。
- 打开历史提问。
- 删除无用会话。

回答应基于认知库引用；证据不足时应明确说明不知道。

## 设置

“设置”管理工作区、认知库路径、分析引擎、模型、飞书 Webhook、自动挖掘、新闻、大事、财报库、商品、DRAM、涨停、持仓会话等配置。

重点配置：

- 认知库路径：每个工作区独立。
- 分析引擎：本机 Claude/Codex/TRAE 或远程 API。
- CLI 运行方式：tmux 或 print。
- 评测引擎：可与主引擎不同，减少自评偏差。
- 最大并发挖掘数：控制本机 CLI 负载。
- 各雷达模块的定时任务和频率。
